# Taskit - Task Management System

A web platform to manage tasks, built with **Next.js 16 (App Router)**, **Tailwind CSS v4**, and **Supabase (PostgreSQL)**.
Features a Jira/Trello-style Kanban dashboard with glassmorphism design, dark mode, and a responsive collapsible sidebar.

## Features

- **Authentication**: Register and Login using a custom `public.users` table with `bcryptjs` password hashing.
- **Kanban Board**: Drag-and-drop tasks between "To Do", "In Progress", and "Done" columns.
- **Nested Subtasks**: Create multi-level subtasks within any task.
- **Effort Estimation**: Aggregates `effort_estimate` values recursively from leaf subtasks to parent tasks.
- **i18n**: English / Spanish language switcher via a custom Language Context (no external library).
- **Themes**: Dark / Light (Sepia) mode, persisted in `localStorage`.
- **Profile**: Edit name and delete account (cascades deletion of all associated tasks).
- **CRUD API**: Full RESTful API for tasks at `/api/tasks` and `/api/tasks/[id]`.

## Database Setup (Supabase)

Create the following tables in the **`public`** schema of your Supabase project.

### `users` table

```sql
create table public.users (
  id bigint generated by default as identity not null,
  name text null,
  email text null,
  password text null,
  created_at timestamp with time zone not null default now(),
  constraint users_pkey primary key (id)
);
```

### `task_list` table

```sql
create table public.task_list (
  id uuid not null default gen_random_uuid(),
  user_id bigint null,
  title character varying null,
  description character varying null,
  status text null,
  priority text null,
  effort_estimate numeric null,
  parent_task_id uuid null,
  created_at timestamp with time zone not null default now(),
  constraint task_list_pkey primary key (id)
);
```

> **Row-Level Security (RLS):** For development, disable RLS on both tables or add permissive policies for the `anon` role. For production, configure proper `auth.uid()` based policies.

## Environment Variables

Create a `.env.local` file in the project root:

```env
NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
```

## Running the Application

Install dependencies:

```bash
npm install
```

Start the development server:

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

---

## Running Unit Tests

Business logic for recursive effort aggregation is tested with **Vitest**:

```bash
npm run test
```

Tests are located in `src/utils/taskHelpers.test.ts` and cover:
- Tree construction and subtask linkage
- Recursive effort aggregation across multiple levels
- `completed_effort` calculation for DONE tasks
- Orphan task handling (unknown `parent_task_id`)
- Empty task list edge case

## API Reference

### Tasks

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/tasks?userId={id}` | Get all tasks for a user |
| `POST` | `/api/tasks` | Create a new task |
| `PUT` | `/api/tasks/[id]` | Update a task |
| `DELETE` | `/api/tasks/[id]` | Delete a task |

### Auth

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/api/auth/register` | Register a new user |
| `POST` | `/api/auth/login` | Login with email & password |

### Users

| Method | Endpoint | Description |
|--------|----------|-------------|
| `PUT` | `/api/users/profile` | Update user name |
| `DELETE` | `/api/users/delete` | Delete account and all associated tasks |

## Deployment (Netlify)

A `netlify.toml` is included with the `@netlify/plugin-nextjs` configuration.
Set the environment variables (`NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`) in your Netlify dashboard under **Site settings → Environment variables**.
